# Google Style B2B Prompt

Use this file for friendly, familiar, low-friction B2B products. Append new `google-style` prompts here.

## Style Intent

Build for broad teams and mixed skill levels. The UI should feel approachable, predictable, forgiving, and easy to learn while staying efficient enough for real operational work.

## Best For

- Internal tools
- Support systems
- Education platforms
- HR and people ops
- CRM for broad teams
- Content operations
- Settings-heavy products

## Core Prompt

Design a friendly internal tool with approachable forms, clear sectioning, soft status indicators, helpful validation, and familiar controls. Keep it operational and efficient, but reduce intimidation for users who are not in the product all day.

Use visible grouping, plain labels, gentle empty states, clear save behavior, and calm feedback. Avoid playful decoration that weakens the product's utility.

## Page Variants

### Settings And Configuration

Design a settings area with grouped sections, clear labels, toggle and select controls, integration status, save/cancel behavior, validation, permissions hints, and a restrained danger zone. Make repeated configuration work predictable and calm.

### Support Workspace

Design a support workspace with queue filters, customer context, conversation status, suggested actions, internal notes, handoff controls, and clear resolution states. Keep it readable and reassuring.

### Data Table Management

Design a data-management page with saved views, search, filters, sortable columns, row selection, bulk actions, pagination, status chips, inline metadata, and a detail drawer or detail page. Include empty, loading, error, and selected-row states when implementing.
