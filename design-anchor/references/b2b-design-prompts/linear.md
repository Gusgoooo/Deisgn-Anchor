# Linear B2B Prompt

Use this file for compact, quiet, high-frequency B2B workbenches. Append new `linear` prompts here.

## Style Intent

Build for users who live in the product all day. Prioritize density, scanning speed, low visual noise, crisp hierarchy, and fast transitions between list, detail, and action.

## Best For

- Operations backend
- CRM workspace
- Task center
- Fulfillment desk
- Project or ticket management
- Data-heavy admin pages
- Developer or internal tools

## Core Prompt

Design a compact B2B operations workbench with a left navigation rail, a high-signal summary strip, saved table views, powerful filters, row status chips, bulk actions, and a right-side detail drawer. Prioritize fast scanning, clear ownership, next actions, and low visual noise.

Use a restrained layout with tight spacing, clear table density, precise typography, subtle borders, semantic status colors, and minimal decoration. The screen should feel built for repeated daily use, not for marketing.

## Page Variants

### Data Table Management

Design a data-management page with saved views, search, filters, sortable columns, row selection, bulk actions, pagination, status chips, inline metadata, and a detail drawer or detail page. Include empty, loading, error, and selected-row states when implementing.

### Approval Queue

Design an approval queue where reviewers can triage by priority, SLA, requester, amount/risk, and missing evidence. Include decision actions, comments, audit trail, escalation, and clear consequences for approve/reject decisions.

### Detail Record

Design a record detail page with a summary header, status, owner, key metadata, primary actions, tabs, related records, timeline/activity, notes, and audit information. Make the page support both quick review and deeper investigation.
